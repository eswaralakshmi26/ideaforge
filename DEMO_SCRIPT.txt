[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at the system architecture. When an idea is submitted, it starts at the Security Checkpoint node, which scrubs PII, checks for prompt injection attempts, and enforces business category guardrails. If a violation is caught, the workflow routes to the Security Event node to block the process. If clean, the idea passes to the orchestrator_agent, which coordinates the work. The orchestrator delegates analysis to our sub-agents: the critic_agent, which calculates the break-even points and analyzes SWOT components; the strategist_agent, which develops the monetization model and unique features; and the marketing_agent, which generates brand names and go-to-market channels. These sub-agents query tools on the FastMCP Server, such as get_market_trends, get_competitor_analysis, calculate_break_even, and generate_brand_assets. Once the reports are collected, the orchestrator compiles them and passes the proposal to the human_review node, which uses a RequestInput event to pause the workflow for human approval or feedback before routing to the post_review_processor and finally to the final_output.
